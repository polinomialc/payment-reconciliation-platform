# Metabase Reporting Demo

This optional demo adds a management-reporting layer on top of the sanitized reconciliation outputs.

Streamlit remains the operational workflow interface. Metabase is intended for managers and leads who need KPI visibility across matched versus open balances, aging exposure, exception backlog, and receipt-side disputes.

## Run Locally

Start Docker Desktop first, then run:

```bash
cd ..
python3 scripts/export_metabase_seed.py
cd metabase
docker compose up -d
```

Open Metabase:

```text
http://localhost:3000
```

During Metabase setup, add a PostgreSQL database connection:

```text
Host: analytics-db
Port: 5432
Database name: metabase
Username: metabase
Password: metabase
Schema: analytics
```

The analytics database is initialized from CSV seeds generated by `scripts/export_metabase_seed.py`, which runs the same DuckDB + SQL pipeline used by the Streamlit demo.

## BI Views

The demo creates these reporting views:

- `analytics.bi_payment_batch_summary`
- `analytics.bi_reconciliation_daily_kpis`
- `analytics.bi_aging_exposure`
- `analytics.bi_exception_backlog`
- `analytics.bi_receipt_exception_summary`
- `analytics.bi_channel_health`

## Suggested Dashboards

- Matched amount by day
- Aging exposure by bucket and operational owner
- Exception backlog by outcome and market
- Chargeback and rejected-transaction trend

## Reset

To rebuild the reporting database from scratch:

```bash
cd ..
python3 scripts/export_metabase_seed.py
cd metabase
docker compose down -v
docker compose up -d
```
